# AI Rules for CV Maker App

## Project Overview
This is a CV Maker web application built with Next.js, Firebase, and Tailwind CSS.
Users fill a form, choose a CV template, preview the result live, save to Firebase, and export as PDF.

## Tech Stack
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS
- **Database**: Firebase Firestore
- **Storage**: Firebase Storage (for user photos)
- **Auth**: Firebase Authentication (email/password)
- **PDF Export**: html2canvas + jsPDF
- **Language**: TypeScript

## Architecture Rules
- Use App Router (`app/` directory), not Pages Router
- All Firebase calls go in `lib/firebase/` — never inline in components
- Use React Server Components where possible; use `"use client"` only when needed
- Keep components small and single-responsibility
- Form state managed with `react-hook-form`
- All CV data typed strictly with TypeScript interfaces defined in `types/cv.ts`

## CV Data Model
All CV profiles stored in Firestore under: `users/{userId}/cvs/{cvId}`

```ts
interface CVData {
  id: string;
  templateId: "template1" | "template2" | "template3";
  updatedAt: Timestamp;

  // Personal
  firstName: string;
  lastName: string;
  email: string;
  phone: string;
  photoURL?: string;
  summary: string;
  visaStatus?: string;
  dob?: string;
  nationality?: string;

  // Skills (bar-based)
  skills: { name: string; level: number }[]; // level 0-100

  // Languages (star-based)
  languages: { name: string; stars: number }[]; // stars 1-5

  // Work Experience
  workExperience: {
    org: string;
    location: string;
    title: string;
    dept?: string;
    start: string;
    end: string;
    duties?: string[];
  }[];

  // Education
  education: {
    school: string;
    degree: string;
    start: string;
    end: string;
  }[];

  // Certifications
  certifications: {
    name: string;
    org: string;
    date: string;
  }[];
}
```

## Template Rules
- Templates are pure HTML/CSS files located in `public/templates/`
- Each template uses `{{placeholder}}` syntax for variable injection
- Template rendering happens client-side by replacing placeholders with CVData values
- Template IDs: `template1` (peach/gray), `template2` (teal), `template3` (dark gray)
- Never hardcode template content in components — always load from `public/templates/`

## Placeholder Convention
All templates use this exact placeholder format: `{{snake_case_name}}`
Examples: `{{first_name}}`, `{{job_1_org}}`, `{{skill_1_level}}`

## Coding Style
- Use `async/await`, never `.then()` chains
- Tailwind only for styling — no inline styles, no CSS modules
- All user-facing strings in English
- Avoid `any` type — be explicit
- Error boundaries around CVPreview component
- Loading skeletons while fetching CV data

## PDF Export
- Triggered by "Download PDF" button in editor
- Use `html2canvas` to capture the CVPreview div
- Pass canvas to `jsPDF` as A4 image
- Filename: `{firstName}_{lastName}_CV.pdf`

## AI Assistant Behavior
- When generating components, always include TypeScript types
- When modifying Firestore logic, always check for existing `lib/firebase/firestore.ts` patterns
- When adding a new template, add its ID to the `templateId` union type in `types/cv.ts`
- Always preserve existing placeholder naming convention when editing templates
- Do not regenerate entire files — make targeted edits
